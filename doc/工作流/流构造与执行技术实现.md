# Flow 构造与执行

## 模块

| 模块 | 职责 |
| --- | --- |
| `gt-flow/lib.rs` | Workflow 领域模型与 YAML 解析 |
| `gt-flow/event.rs` | CloudEvents 目录、匹配和运行意图 |
| `gt-flow/node.rs` | 空注册表、所有者、原子替换和节点编译 |
| `gt-flow/builder.rs` | 构造草稿与诊断 |
| `gt-flow/runner.rs` | 表达式渲染、顺序执行和运行报告 |
| `commands/flow.rs` | 来源巡检、Core/插件执行分派和 Tauri API |

## 注册表重建

`FlowNodeRegistry::new()` 创建空池。宿主巡检文件、Git、远程和数据同步 Core provider，
再合并 `ExtensionRegistry` 中全部活跃插件的 `flowNodes`，构造完整 candidate。所有校验
成功后一次性替换当前池。

节点池属于派生状态，权威来源只有 Core provider 和活跃插件 manifest。插件市场不再
手工维护某一个节点，而是在生命周期变化后触发完整重建。

## 运行

```text
读取 Flow -> clone 节点池 -> 固定插件 binding -> 编译节点
         -> 渲染 inputs -> Core executor / plugin sidecar
         -> 合并 outputs -> FlowRunReport -> 发布运行事件
```

Flow 在 blocking 线程执行，并由全局执行锁避免并发写仓库。插件节点只收到已渲染输入和
宿主筛选的运行上下文；返回值必须符合 `{ outputs, skipped, message }`。

## 失败边界

- 未注册节点：编译为 unknown，运行失败。
- 节点输入或表达式错误：当前节点失败，后续节点停止。
- 插件快照变化：返回 `extension_changed_during_flow_node`。
- 第二个并发运行：返回 `flow_run_already_in_progress`。
