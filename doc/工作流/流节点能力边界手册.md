# Flow 节点能力边界

## 原则

- Core 节点必须对应当前可执行的基础设施能力。
- 领域能力属于插件，例如站点扫描和构建属于 `site-publisher`。
- 未实现的计划不进入注册中心。
- 节点定义、执行 handler 和测试必须同时存在。
- 插件节点 `uses` 必须位于自己的插件 ID 命名空间。

## Core

| 能力 | 边界 |
| --- | --- |
| 文件存在校验 | 检查本地文件或目录，可选检查非空 |
| 目录同步 | 递归复制；当前不删除目标孤儿文件 |
| Git 提交 | stage all + commit；无变更时 skipped |
| Git 推送 | 使用宿主认证配置推送指定 remote/branch |
| 数据同步 | 执行数据中心 pull/import/export/commit/push |

## 插件

发布插件当前提供扫描和构建节点。构建节点接受仓库路径、输出目录、站点标题和 include，
调用真实 `na_site::build_site`，返回页面数、资源数和链接检查结果。

插件节点的 method 必须在 `backend.methods` 声明。安装、更新和卸载都会完整重建节点池；
卸载后既有 Flow 定义保留，但相关步骤变为未注册。

## 暂不注册

通知、自动备份、发布上下文适配等能力在没有真实 handler 前不注册。Flow 运行结果本身负责
表达成功、失败和节点 message，不用伪通知节点代替。
