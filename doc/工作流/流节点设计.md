# Flow 节点设计

## 模型

节点定义描述 `uses`、名称、类型、输入 schema 和输出 schema。注册记录额外保存所有者：

```text
Core
Plugin(pluginId)
```

`uses` 是稳定执行标识；显示名称和插件版本只用于目录展示。Flow YAML 保存 `uses`，不保存
动态库路径或 backend method。

## 贡献

Core provider 与具体能力模块相邻，避免 Flow 引擎维护一份可能漂移的总清单。插件通过：

```json
{
  "contributes": {
    "flowNodes": [{
      "uses": "com.example.plugin/action@v1",
      "name": "Action",
      "type": "action",
      "summary": "Run action",
      "inputs": { "path": "string" },
      "outputs": { "count": "number" },
      "method": "flow.action"
    }]
  }
}
```

输入类型后缀 `?` 表示可选字段。插件 method 必须在 `backend.methods` 中声明。

## 生命周期

节点池不做增量修补。每次来源变化都从 Core provider 和活跃插件重新构建 candidate，成功后
原子替换。冲突或非法 manifest 不得产生部分注册结果。

## 执行

Core owner 进入宿主 Core executor；Plugin owner 根据运行开始时保存的 binding snapshot 进入
sidecar。目录展示和执行使用同一所有者信息，不通过字符串前缀推断。
