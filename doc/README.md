# doc/

本目录存放项目文档,按模块分子目录组织。新增文档时先找归属模块子目录,不要直接放在 `doc/` 根目录。

## 目录结构与归档规则

```
doc/
├── README.md                    ← 全局索引(本文件,唯一允许存在于根目录的文档)
├── 架构/                        ← 跨模块总览、协作规范、环境与排障
│   ├── 项目模块现状.md          ← 当前仓库模块与代码状态总览(先看这个)
│   ├── 代码提交规范.md
│   └── 开发调试与启动指南.md
├── Git/                         ← gt-git 模块设计
│   └── Git基础能力设计.md
├── 数据引擎/                    ← gt-store 模块设计
│   ├── 数据中心设计.md
│   ├── 数据中心远程同步设计.md
│   ├── 数据流与响应链路规范.md
│   └── 配置目录规范.md
├── 工作流/                      ← gt-flow 模块设计
│   ├── 流体系设计规范.md
│   ├── 流节点设计.md
│   ├── 流构造与执行技术实现.md
│   ├── 工作流操作说明手册.md
│   ├── 流节点能力边界手册.md
│   ├── Flow节点使用速查与片段.md
│   └── 事件池.md
├── 站点/                        ← gt-site 模块设计(含 Pages 发布)
│   ├── 静态HTML构建器设计.md
│   ├── Pages发布目标设计.md
│   └── Pages发布落地计划.md
├── UI/                          ← 前端视觉与组件规范
│   └── 全局UI规范.md
├── 开发日志/                    ← 按时间顺序的开发过程记录
│   ├── 开发记录-01-UI基建与插件Shell.md
│   └── 重构记录.md              ← src-tauri/src/lib.rs 命令层拆分重构
└── 杂项/                        ← 不属于任何模块的零散排障笔记
    └── Tauri-DMG-打包卡在-Finder-美化脚本.md
```

**归档规则(新增文档前先看这里):**

1. 文档属于某个 Rust crate / 前端插件模块的设计或实现记录 → 放进对应模块子目录(如 `gt-git` 相关放 `Git/`)。
2. 文档跨模块、描述整体架构/现状/协作规范/环境排障 → 放进 `架构/`。
3. 文档记录某次开发过程的决策与踩坑(时间线形式) → 放进 `开发日志/`,文件名带序号前缀。
4. 文档是一次性的、和长期模块设计无关的排障笔记 → 放进 `杂项/`。
5. 新模块出现时,新建一个以模块名命名的子目录(参照 `站点/` 的命名方式),不要堆在根目录。
6. 每次新增或移动文档,回到本文件更新下面的索引表和目录树。
