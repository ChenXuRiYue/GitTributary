# 插件顶栏规范 (PluginHeader 三段式)

> 本文是全项目插件主视图顶栏的统一约定，是 `doc/UI/全局UI规范.md` 「插件 Shell」与「挑动内容与边缘控制」两节的落地细化。
> 所有全高插件 (Git / 文档发布 / 数据中心 / Flow / 扩展) 的顶栏都应遵循本约定。
> 首个参考实现: 文档发布插件 `src/plugins/site/SitePanel.tsx` 的 `<header>`。

---

## 一、为什么要统一顶栏

历史问题:

- 顶栏信息啰嗦 —— 徽章 + 标题 + 上下文名 + 描述 + 大按钮堆在一起，视觉噪音大。
- 可变内容 (仓库名、任务名、状态) 直接贴边或占大块，状态一变就左右弹跳。
- 每个插件各写各的，风格不一致。

目标: 一条**安静、稳定、一行**的顶栏，信息密度服务于操作，不因选中项/状态变化而抖动。

---

## 二、三段式结构

顶栏在水平方向分为三段，单行排布:

```text
┌──────────────────────────────────────────────────────────────────────┐
│ [图标] 标题 · <上下文选择器 ▾>                        [次操作] [主操作] │
│  左段: 身份 (稳定)      中段: 上下文 (可变、可截断)      右段: 操作     │
└──────────────────────────────────────────────────────────────────────┘
```

| 段 | 内容 | 稳定性 | 规则 |
| --- | --- | --- | --- |
| **左段 · 身份** | 插件图标 + 插件标题 (`gt-title-app`) | 恒定不变 | `shrink-0`，永远不截断、不换行 |
| **中段 · 上下文** | 当前操作对象的选择器 (任务 / 仓库 / Profile 等)，名称 + `▾` | 随选择变化 | `min-w-0` + `truncate` + `max-w`，只在这里截断，变化不影响左右段位置 |
| **右段 · 操作** | 主操作 (高亮) + 至多 1~2 个次操作 | 相对稳定 | `ml-auto` / `justify-between` 靠右；按钮数量克制 |

三段之间用一个浅色 `·` 分隔左段与中段；右段用 `ml-auto` 推到最右。

---

## 三、硬性规则

1. **单行**: 顶栏 `flex items-center`，不用 `flex-wrap`。高度约 `py-3` (≈52px)，不做两行。
2. **标题稳定**: 左段图标 + 标题固定，`shrink-0`；不把仓库名/任务名等可变文本放进左段。
3. **可变内容只在中段**: 上下文选择器用 `min-w-0 truncate max-w-[16rem]`，并带 `title` 全称提示；名称再长也只在中段收缩，不挤压左右。
4. **不堆冗余标签**: 禁止 "身份徽章 + 标题 + 同名上下文" 这种重复表达 (例如旧版 `[工作区]徽章 + 文档发布 + 工作区名` 三处近义)。上下文只出现一次。
5. **不放描述行**: 顶栏不放 `description` 段落；说明性文案下沉到面板内。
6. **右段克制**: 右侧固定操作不超过 2~3 个，避免形成挤压中段的 "按钮墙"。低频配置收进选择器菜单或二级工具区。
7. **语义 token + gt-* 字号**: 标题 `gt-title-app`，上下文名 `gt-body-strong`，路径/副信息 `gt-caption`；颜色走语义 token，禁止硬编码。

---

## 四、中段选择器 (上下文切换器) 约定

中段的选择器既是 "当前对象展示"，也是 "切换入口"，点击展开一个菜单:

- 触发器: `variant="ghost"`、`h-8`、`max-w-[16rem]`，内容为 `名称 + ChevronDown`，`aria-haspopup="menu"` / `aria-expanded`。
- 菜单: 左对齐 (`left-0`) 挂在触发器下方，`z-30`，`rounded-md border bg-popover shadow-lg`。
- 菜单结构 (自上而下):
  1. 头部: 当前对象名 + 副信息 (如源仓库路径) + 一个 "设置" 快捷入口。
  2. 列表: 可选对象，当前项高亮 (`bg-primary/8` + Check 图标)。
  3. 底部: "新建" + "设置/配置" 两个操作。
- 关闭: 点击外部 (`pointerdown`) 或 `Escape`。

> 切换与配置分离: 菜单只做**选择 + 新建 + 进入配置**；对象的详细编辑放到对应视图，不在菜单里做重表单。

## 五、参考实现 (文档发布)

`SitePanel.tsx` 顶栏:

```tsx
<header className="border-border flex shrink-0 items-center gap-2 border-b px-7 py-3">
  <div className="flex size-8 shrink-0 items-center justify-center rounded-md bg-primary/10 text-primary">
    <BookOpenCheck className="size-[18px]" />
  </div>
  <h2 className="gt-title-app shrink-0">文档发布</h2>
  <span className="shrink-0 text-muted-foreground">·</span>
  <div className="relative min-w-0">
    <Button variant="ghost" className="h-8 max-w-[16rem] justify-between gap-2 px-2" title={taskLabel}>
      <span className="min-w-0 truncate gt-body-strong">{taskLabel}</span>
      <ChevronDown className="size-3.5 shrink-0 text-muted-foreground" />
    </Button>
    {/* 展开: 任务切换菜单 (头部/列表/底部三段) */}
  </div>
</header>
```

- 左段: 图标 + "文档发布"，恒定。
- 中段: "发布任务" 选择器，可截断，点击切换/新建/进设置。
- 右段: 当前暂无常驻主操作 (构建/发布仍在面板内)；待读写分离重构后，右段可挂 "发布" 主操作。

## 六、其它插件套用建议

| 插件 | 左段标题 | 中段上下文 | 右段主操作 |
| --- | --- | --- | --- |
| Git | Git | 当前仓库 ▾ | 提交 / 同步 |
| 文档发布 | 文档发布 | 发布任务 ▾ | (重构后) 发布 |
| 数据中心 | 数据中心 | Profile / 环境 ▾ | 同步 |
| Flow | 工作流 | 当前流 ▾ | 运行 |

各插件按此模型套用，保证跨插件顶栏一致、安静、不抖动。

---

## 七、Do / Don't

**Do**
- 单行、三段、标题稳定、上下文只出现一次。
- 可变文本只在中段截断，带 `title` 全称。
- 右段操作克制，低频项收进菜单。

**Don't**
- ❌ 身份徽章 + 标题 + 同名上下文 三处重复。
- ❌ 顶栏放描述段落 / 两行堆叠。
- ❌ 把仓库名/任务名等可变长文本贴左缘或占大块。
- ❌ 右侧堆一排固定按钮挤压中段。
