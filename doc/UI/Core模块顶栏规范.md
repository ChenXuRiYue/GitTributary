# Core 模块顶栏规范 (DomainTrail)

> 本文约束 Git、数据、Flow 等 Core 主视图及工作台插件页面的顶栏。
> 首个参考实现：`plugins/site-publisher/frontend/src/site/SitePanel.tsx` 的 `<header>`。

---

## 一、为什么要统一顶栏

历史问题:

- 顶栏信息啰嗦 —— 徽章 + 标题 + 上下文名 + 描述 + 大按钮堆在一起，视觉噪音大。
- 可变内容 (仓库名、任务名、状态) 直接贴边或占大块，状态一变就左右弹跳。
- 每个工作台页面各写各的，风格不一致。

目标: 一条**安静、稳定、一行**的顶栏，只表达最上层资源坐标: 当前能力域、当前页面域、当前资源上下文。保存、发布、运行、提交等业务操作必须下沉到页面或面板内。

---

## 二、顶栏结构

顶栏在水平方向分为两栏，单行排布。左栏是资源坐标，从左到右增长；右栏是域统计，从右到左排布:

```text
┌──────────────────────────────────────────────────────────────────────┐
│ [一级域 / 二级域 / 当前资源 ▾]        [二级域统计 / 一级域统计]        │
│  左栏: 资源坐标 LTR                         右栏: 域统计 RTL          │
└──────────────────────────────────────────────────────────────────────┘
```

| 栏 | 内容 | 方向 | 规则 |
| --- | --- | --- | --- |
| **左栏 · 资源坐标** | `DomainTrail` + 当前资源对象选择器，例如 `文档发布 / 文档范围 / 发布任务 ▾` | LTR | 固定在顶栏最左；资源对象无固定字段标签和外框；长文本只在资源对象内截断 |
| **右栏 · 域统计** | 一级域统计 + 当前二级域统计，例如 `候选:86 已选:18 md:120 / 任务:3 已配:2` | RTL | 每个统计项使用 `名:值`，同组内用空格分隔，域组之间使用弱化的 `/`；只放聚合计数，不放页面动作 |

左栏保持朴素路径表达，不使用图标、徽章、键帽或额外装饰。右栏只允许域级统计信息，不明文展示“一级域/二级域”等结构标签。顶栏不设置业务操作区。

---

## 三、多级域标题组件 `DomainTrail`

`DomainTrail` 是工作台顶栏最左侧的标准定位组件，路径为:

```text
src/components/DomainTrail.tsx
```

它接收一个从粗到细的域数组:

```tsx
<DomainTrail
  items={[
    { id: "site", label: "文档发布" },
    { id: "capture", label: "文档范围" },
  ]}
/>
```

视觉规则:

- 顶栏域标题不得复用侧边栏图标，避免同一语义符号在相邻区域重复出现。
- 默认不额外创造图形标记。层级感只用文字权重、颜色和 `/` 表达。
- 一级域使用较高字重，表达工作台能力域，例如 `文档发布`。
- 二级域使用次级字重与弱化颜色，表达当前模块内视图，例如 `文档范围`。
- 分隔符固定使用 `/`，两侧留窄间距；不使用图标、箭头、徽章、键帽或下标。
- 最多展示 2～3 级。超过 3 级时应把更细的对象放入左栏末尾的资源上下文选择器，而不是继续拉长域路径。
- `DomainTrail` 只表达“我在哪个能力域”，不承载任务名、仓库名、URL、错误提示等可变对象。

## 四、硬性规则

1. **单行**: 顶栏 `flex items-center`，不用 `flex-wrap`。高度约 `py-3` (≈52px)，不做两行。
2. **坐标稳定**: 左栏先放 `DomainTrail`，再放当前资源上下文选择器；不复用侧边栏图标。
3. **可变资源只在上下文选择器**: 上下文选择器用 `min-w-0 truncate max-w-[16rem]`，并带 `title` 全称提示；名称再长也只在选择器内收缩，不挤压域路径。
4. **不堆冗余标签**: 禁止 "身份徽章 + 标题 + 同名上下文" 这种重复表达 (例如旧版 `[工作区]徽章 + 文档发布 + 工作区名` 三处近义)。上下文只出现一次。
5. **不放描述行**: 顶栏不放 `description` 段落；说明性文案下沉到面板内。
6. **禁止业务操作**: 顶栏不放保存、发布、运行、提交、删除、同步等页面动作，也不放当前页面的保存状态。操作归属到页面工具区、面板头部或底部操作区。
7. **统计只放聚合计数**: 右栏统计只放一级域/二级域的稳定聚合信息，例如任务数、已配置数、候选数、页面数、变更数；不放“未保存/已保存/待处理”这类页面状态。
8. **语义 token + gt-* 字号**: 标题 `gt-title-app`，上下文名 `gt-body-strong`，路径/副信息 `gt-caption`；颜色走语义 token，禁止硬编码。

## 五、资源上下文选择器约定

资源上下文选择器既是 "当前资源对象展示"，也是 "切换入口"，点击展开一个菜单:

- 触发器: `variant="ghost"`、`h-8`、`max-w-[16rem]`，内容为 `名称 + ChevronDown`，`aria-haspopup="menu"` / `aria-expanded`。
- 菜单: 左对齐 (`left-0`) 挂在触发器下方，`z-30`，`rounded-md border bg-popover shadow-lg`。
- 菜单结构 (自上而下):
  1. 头部: 当前对象名 + 副信息 (如源仓库路径) + 一个 "设置" 快捷入口。
  2. 列表: 可选对象，当前项高亮 (`bg-primary/8` + Check 图标)。
  3. 底部: "新建" + "设置/配置" 两个操作。
- 关闭: 点击外部 (`pointerdown`) 或 `Escape`。

> 切换与配置分离: 菜单只做**选择 + 新建 + 进入配置**；对象的详细编辑放到对应视图，不在菜单里做重表单。
> 页面动作分离: 保存范围、发布、运行等动作必须出现在产生该动作的页面或面板中，不得进入 Core 顶栏。
> 查看与当前分离: 配置视图中的对象列表点击只表示“查看/编辑详情”，不得隐式改变顶栏资源坐标中的当前上下文；切换当前上下文必须使用明确命令，例如“设为当前任务”或顶栏资源选择器。

## 六、参考实现 (文档发布)

`SitePanel.tsx` 顶栏:

```tsx
<header className="border-border flex shrink-0 items-center gap-2 border-b px-5 py-2">
  <div className="flex min-w-0 flex-1 items-center gap-1.5">
    <DomainTrail
      items={[
        { id: "site", label: "文档发布" },
        { id: activeView.id, label: activeView.name },
      ]}
    />
    <span className="text-muted-foreground/60 gt-body">/</span>
    <div className="relative min-w-0 shrink">
      <button type="button" className="flex h-7 max-w-[16rem] min-w-0 items-center gap-1.5 rounded px-1.5 text-muted-foreground" title={taskLabel}>
        <span className="min-w-0 truncate gt-body">{taskLabel}</span>
        <ChevronDown className="size-3.5 shrink-0" />
      </button>
      {/* 展开: 任务切换菜单 (头部/列表/底部三段) */}
    </div>
  </div>
  <div className="ml-auto hidden items-center gap-2 md:flex">
    <span>候选:86 已选:18 md:120</span>
    <span className="text-muted-foreground/40">/</span>
    <span>任务:3 已配:2</span>
  </div>
</header>
```

- 左栏: 朴素坐标表达，例如 `文档发布 / 文档范围 / 个人知识库发布任务 ▾`。
- 右栏: 只显示统计值，例如 `候选:86 已选:18 md:120 / 任务:3 已配:2`；不显示“一级域/二级域”结构标签。
- 页面动作: 文档范围页的“保存范围”放在“发布范围”面板头部，不进入顶栏。

## 七、其它工作台页面套用建议

| 插件 | 左栏资源坐标 | 右栏统计 | 页面动作归属 |
| --- | --- | --- | --- |
| Git | `Git / 变更 / 当前仓库 ▾` | 仓库数 / 文件变更数 | 提交 / 同步放变更面板 |
| 文档发布 | `文档发布 / 文档范围 / 发布任务 ▾` | 任务数 / 候选与已选数 | 保存范围放发布范围面板，构建/发布放发布执行页 |
| 数据中心 | `数据 / 同步 / Profile ▾` | Profile 数 / 同步对象数 | 同步放页面工具区 |
| Flow | `流 / 运行 / 当前流 ▾` | 流数量 / 节点或运行记录数 | 运行放执行面板 |

各插件按此模型套用，保证跨插件顶栏一致、安静、不抖动。

## 八、Do / Don't

**Do**
- 单行、两栏，左栏 LTR 做资源坐标，右栏 RTL 做域统计。
- 用朴素路径表达域标题，避免散开的图标、徽章和标签堆。
- 可变文本只在资源上下文选择器内截断，带 `title` 全称。
- 右栏只展示 `名:值` 聚合计数，不展示结构标签或页面状态。
- 页面动作和保存状态放进页面内部，与被操作对象保持内聚。

**Don't**
- ❌ 身份徽章 + 标题 + 同名上下文 三处重复。
- ❌ 把二级域标题放到内容区再造一条页面级顶栏。
- ❌ 顶栏放“保存范围 / 发布 / 运行 / 提交 / 同步”等业务操作。
- ❌ 顶栏放当前页面的未保存、已保存、待处理等状态。
- ❌ 右栏明文展示“一级域 / 二级域”这类设计结构词。
- ❌ 右栏统计从左往右堆，导致一级域统计不贴近右侧锚点。
- ❌ 顶栏放描述段落 / 两行堆叠。
- ❌ 把仓库名/任务名等可变长文本贴左缘或占大块。
- ❌ 右侧堆一排固定按钮挤压资源上下文。
