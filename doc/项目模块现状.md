## 总体架构

Git Tributary 当前是一个 Tauri 2 桌面应用:

```text
React / TypeScript / Vite / Tailwind
        |
        | Tauri invoke
        v
src-tauri/src/lib.rs
        |
        +-- gt-git   Git 能力层,基于 git2
        +-- gt-store 数据中心,JSONL + Profile + 同步配置
```

前端已经形成插件化 Shell:

- 一级插件由 `src/plugins/registry.ts` 注册。
- Git 面板内部有二级视图注册表 `src/plugins/git/registry.ts`。
- 通用导航组件 `IconNav` 同时复用于一级插件栏和 Git 二级栏。
- UI 基础组件位于 `src/components/ui/`,主要是手写 shadcn 风格组件。

后端已经形成两个核心 crate:

- `gt-git`: 仓库打开、状态、diff、提交、分支、历史、远程操作。
- `gt-store`: 配置 KV、命名空间、Profile、workspace 状态、凭证、同步配置。

## Git 模块

### 前端视图

Git 面板是“二级插件”结构,当前注册了 6 个视图:

| 视图 | 文件 | 当前状态 | 已实现内容 |
| --- | --- | --- | --- |
| 变更 | `ChangesView.tsx` | 可用 | 打开仓库、自动恢复最近仓库、刷新状态、文件勾选、查看 diff、全部/选择性提交 |
| 历史 | `HistoryView.tsx` | 可用 | 查看提交列表、悬浮预览提交详情、选择提交、查看提交文件与单文件 diff |
| 分支 | `BranchesView.tsx` | 可用 | 本地分支列表、创建、切换、删除 |
| 远程 | `RemoteView.tsx` | 可用 | 远程列表、添加/删除远程、fetch/pull/push、保存项目级 token |
| 安全 | `SafetyView.tsx` | 可用 | Git 用户名/邮箱/远程 URL、公共 token、SSH key 路径保存与掩码展示 |
| 自动 | `AutoView.tsx` | 占位 | 展示“自动备份”方向,尚未接定时提交、文件监听、规则配置 |

### 后端能力

`gt-git` 当前已经实现:

| 能力域 | 代码位置 | 已实现接口 |
| --- | --- | --- |
| 仓库生命周期 | `repo.rs` | `open`, `init`, `is_repo`, `workdir`, `overview` |
| 状态 | `status.rs` | 当前分支、工作区状态、文件状态结构 |
| diff | `diff.rs` | 工作区文件 diff、提交文件 diff |
| 提交 | `commit.rs` | `stage_all`, `stage_files`, `commit` |
| 分支 | `branch.rs` | 本地分支列表、创建、切换、删除 |
| 历史 | `log.rs` | 提交历史、分支历史、提交文件列表 |
| 远程 | `remote.rs` | 远程列表、添加、修改 URL、删除、fetch、push、fast-forward pull |
| 认证 | `remote.rs` + Tauri command | Token、SSH key、SSH agent、None 的认证回调 |

Tauri command 层在 `src-tauri/src/lib.rs` 暴露了这些能力,并在打开仓库、切换分支后调用 `store.sync_workspace(...)` 同步当前 workspace 状态。

### 已知缺口

- 还没有事件总线,Git 操作完成后前端仍主要靠手动 refresh 或视图初始化读取。
- 还没有文件监听、定时自动提交、自动 commit message、危险操作前快照。
- `pull` 只支持 fast-forward,遇到分叉会报错并要求手动合并。
- 历史数据挖掘能力还未实现,例如文件最后修改时间 map、提交频率、文件提交次数。
- Git 二级视图已经注册表化,但还不是外部插件动态加载。

## 数据中心模块

### 前端能力

`StorePanel` 当前已经实现:

- 左侧列出所有命名空间、条目数量、public/private 可见性。
- 支持 Profile 下拉切换。
- 右侧查看命名空间下所有 KV 条目。
- 支持 key 搜索、删除单个 key、compact 命名空间。
- 对 `git.access_token`、`git.ssh_passphrase`、`project.*.token` 做 L0 完全掩码。

### 后端能力

`gt-store` 当前已经实现:

| 能力域 | 代码位置 | 已实现内容 |
| --- | --- | --- |
| KV 存储 | `store.rs`, `namespace.rs`, `record.rs` | JSONL append-only 存储、get/set/delete/keys/entries/scan/history/compact |
| 命名空间 | `store.rs` | 自动加载 `data/*.jsonl`,按命名空间推断 public/private |
| Profile | `store.rs` | list/active/switch/create/delete,切换时覆盖 `settings` |
| Workspace | `workspace.rs` | active repo、active branch、recent repos、device id/name、聚合同步函数 |
| Credentials | `credentials.rs` | Git 用户名、邮箱、远程 URL、token、SSH key,敏感信息存 private namespace |
| Sync | `sync.rs` | 同步配置、同步状态、初始化配置仓库、commit/push/pull/sync |

Tauri command 层已经暴露:

- `store_get`, `store_set`, `store_delete`, `store_keys`, `store_entries`, `store_scan`, `store_compact`
- `store_list_profiles`, `store_active_profile`, `store_switch_profile`, `store_create_profile`, `store_delete_profile`
- `get_workspace_info`, `get_recent_repos`
- `sync_get_config`, `sync_set_config`, `sync_now`, `sync_get_state`
- `get_git_credentials`, `set_git_username`, `set_git_email`, `set_git_remote_url`, `set_git_token`, `clear_git_token`, `set_git_ssh_key`

### 已知缺口

- `StorePanel` 目前没有新增/编辑 KV 的 UI。
- Profile 只能切换,前端还没有创建/删除 Profile 的入口。
- sync command 已有,但前端还没有对应配置和触发界面。
- `useStoreKey` 已存在,但还没有配合 Tauri event 做真正的响应式订阅。
- private 数据目前通过命名空间分层和掩码展示保护,尚未实现加密存储。

## UI 与插件 Shell

当前 UI 基建已经完成:

- `App.tsx`: 左侧插件侧边栏 + 右侧面板,支持收起、展开、拖拽调宽、系统区固定到底部。
- `IconNav.tsx`: 图标导航,支持 pinned/overflow/system 分组。
- `DiffPanel.tsx` + `DiffViewer.tsx`: Git 变更和历史共用的 diff 展示组件。
- `ResizeHandle.tsx`: 历史视图的左右区域拖拽分隔条。
- `BrandIcon.tsx`, `BrandSplash.tsx`: 品牌相关组件。
- `src/index.css`: 全局 Tailwind v4、语义 token、基础主题和玻璃效果。

插件抽象当前是“静态注册表”:

```text
PluginDescriptor
  id
  name
  description
  icon
  panel
  category
  pinned
```

已经能支撑内部模块扩展,但还不是 manifest、WASM 或外部包形式的插件系统。

## 当前实现成熟度

| 层级 | 成熟度 | 说明 |
| --- | --- | --- |
| 桌面 Shell | 中 | 可运行、布局清晰、可扩展,但布局状态未持久化 |
| Git 变更/提交 | 中高 | 已能打开仓库、看状态、看 diff、选择性提交 |
| Git 历史 | 中 | 已能看 log 和提交 diff,缺历史挖掘与过滤能力 |
| Git 分支/远程 | 中 | 基础操作可用,复杂合并/冲突处理未做 |
| 数据中心 | 中 | JSONL KV、namespace、Profile、workspace 已可用,前端编辑能力不足 |
| 凭证安全 | 中 | public/private 分层与掩码已做,加密和系统钥匙串未做 |
| 自动备份 | 低 | UI 占位,后端基础 Git 能力已具备 |
| 复习 | 低 | UI 占位,依赖的 Git 数据挖掘未实现 |
| AI | 低 | UI 占位,没有模型接入与上下文管线 |
| 外部插件生态 | 低 | 内部静态注册表已做,动态插件系统未落地 |

## 建议下一步

1. 补齐数据中心前端写操作:新增/编辑 KV、创建/删除 Profile、同步配置面板。
2. 做 Git 自动模块 M1:文件监听、定时检测 dirty、生成固定格式 commit message、commit 后可选 push。
3. 做 store 事件推送:后端 `store_set` 后 emit,前端 `useStoreKey` 监听,减少手动 refresh。
4. 为复习模块补 Git 数据挖掘接口:按文件查询最后修改时间、提交次数、活跃度。
5. 把危险操作统一加确认与快照策略,例如删除分支、pull 前 dirty 检查、push 前敏感信息检查。
