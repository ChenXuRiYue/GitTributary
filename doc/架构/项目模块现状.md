## 总体架构

Git Tributary 当前是一个 Tauri 2 桌面应用:

```text
React 19 / TypeScript / Vite 7 / Tailwind v4
        |
        | Tauri invoke
        v
src-tauri/src/lib.rs
        |
        +-- gt-git   Git 能力层,基于 git2
        +-- gt-store 数据中心,JSONL + Profile + 远程同步
        +-- gt-flow  工作流引擎,GitHub Actions 子集 + CloudEvents
        +-- gt-files 受限路径下的文件列举、扫描、检索和文本读取
```

前端已经形成工作台 Shell:

- Git、Flow、数据、文件管理和插件管理属于 Core；发布位于 `plugins/site-publisher/`。
- Core 注册表与插件 view 都会转换为统一 `WorkbenchItem`，侧边栏不判断来源。
- Core 项静态注册，不参与插件安装和卸载。
- Git 面板内部的视图注册表只用于组织内建视图。
- 通用导航组件 `IconNav` 复用于工作台项和 Git 视图导航。
- UI 基础组件位于 `src/components/ui/`,主要是手写 shadcn 风格组件。

后端已经形成四个 Core 核心包:

- `gt-git`: 仓库打开、状态、diff、提交、分支、历史和远程操作。
- `gt-store`: 配置 KV、命名空间、Profile、workspace 状态、凭证、远程同步(SyncEngine)。
- `gt-flow`: 工作流解析、事件池、支持 Core/插件所有者的节点注册表、Flow 构造草稿、顺序 Runner。
- `gt-files`: 受限路径校验，以及目录列举、递归扫描、文件检索和文本读取。

Tauri command 层在 `src-tauri/src/lib.rs` 暴露了这些能力,并在打开仓库、切换分支后调用 `store.sync_workspace(...)` 同步当前 workspace 状态。

### 已知缺口

- Git 写操作已开始向 Flow 事件池发布 CloudEvent（如 commit、push）,但前端仍主要靠手动 refresh 或视图初始化读取,尚未形成完整响应式订阅。
- 还没有文件监听、定时自动提交、自动 commit message、危险操作前快照。
- `pull` 只支持 fast-forward,遇到分叉会报错并要求手动合并。
- 历史数据挖掘能力还未实现,例如文件最后修改时间 map、提交频率、文件提交次数。
- Git 视图注册表是 Core 内部组织方式，不属于插件体系。

## 工作台 Shell 与插件

当前 UI 基建已经完成:

- `App.tsx`: 左侧工作台项 + 右侧内容，支持收起、展开、拖拽调宽、系统区固定到底部。
- `IconNav.tsx`: 图标导航,支持 pinned/overflow/system 分组。
- `DiffPanel.tsx` + `DiffViewer.tsx`: Git 变更和历史共用的 diff 展示组件。
- `FileTree.tsx`: 文件树组件。
- `ResizeHandle.tsx`: 历史视图的左右区域拖拽分隔条。
- `BrandIcon.tsx`, `BrandSplash.tsx`: 品牌相关组件。
- `src/index.css`: 全局 Tailwind v4、语义 token、基础主题和玻璃效果。

当前架构分为三层：

```text
Core     -> src/core/ 静态注册 -> WorkbenchItem
核心包   -> gt-git / gt-flow / gt-store / gt-files，编译期链接
插件     -> manifest view/flowNode -> WorkbenchItem/Flow 注册表 -> Extension API/sidecar
                              -> gt-plugin-host + Rust cdylib
```

其中 `gt-files` 通过 `files.list`、`files.scan`、`files.search`、`files.readText`
向插件提供受控文件能力；Markdown 候选、渲染与 Pages 发布不属于 Core。

只有第三层称为插件。插件目前来自根目录 `plugins/`，正式构建时进入安装包；支持运行时安装、更新和卸载。
当前仍是 MVP，不支持远程下载和第三方发布。

## 建议下一步

1. 补齐数据中心前端写操作:新增/编辑 KV、同步状态展示面板。
2. 做 Git 自动模块 M1:文件监听、定时检测 dirty、生成固定格式 commit message、commit 后可选 push。
3. 补齐 store 事件前端订阅:后端 `store_set` 已发 Flow 事件,前端 `useStoreKey` 还需要监听 Tauri 事件来减少手动 refresh。
4. 为复习模块补 Git 数据挖掘接口:按文件查询最后修改时间、提交次数、活跃度。
5. 把危险操作统一加确认与快照策略,例如删除分支、pull 前 dirty 检查、push 前敏感信息检查。
6. Flow 自动执行接入:schedule 定时器、file_watch 文件监听,以及 store/git 事件匹配后的后台运行。
