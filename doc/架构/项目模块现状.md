## 总体架构

Git Tributary 当前是一个 Tauri 2 桌面应用:

```text
React 19 / TypeScript / Vite 7 / Tailwind v4
        |
        | Tauri invoke
        v
src-tauri/src/lib.rs
        |
        +-- gt-git   Git 能力层,基于 git2
        +-- gt-store 数据中心,JSONL + Profile + 远程同步
        +-- gt-flow  工作流引擎,GitHub Actions 子集 + CloudEvents
        +-- gt-site  静态站点构建与发布
```

前端已经形成插件化 Shell:

- 一级插件由 `src/plugins/registry.ts` 注册。
- Git 面板内部有二级视图注册表 `src/plugins/git/registry.ts`。
- 通用导航组件 `IconNav` 同时复用于一级插件栏和 Git 二级栏。
- UI 基础组件位于 `src/components/ui/`,主要是手写 shadcn 风格组件。

后端已经形成四个核心 crate:

- `gt-git`: 仓库打开、状态、diff、提交、分支、历史、远程操作、Pages 发布辅助。
- `gt-store`: 配置 KV、命名空间、Profile、workspace 状态、凭证、远程同步(SyncEngine)。
- `gt-flow`: 工作流解析、事件池、节点注册表、Flow 构造草稿、顺序 Runner。
- `gt-site`: 静态站点扫描、Markdown→HTML 构建、发布产物准备。

## Git 模块

### 前端视图

Git 面板是"二级插件"结构,当前注册了 6 个视图:

| 视图 | 文件 | 当前状态 | 已实现内容 |
| --- | --- | --- | --- |
| 变更 | `ChangesView.tsx` | 可用 | 打开仓库、自动恢复最近仓库、刷新状态、文件勾选、查看 diff、全部/选择性提交 |
| 历史 | `HistoryView.tsx` | 可用 | 查看提交列表、悬浮预览提交详情、选择提交、查看提交文件与单文件 diff |
| 分支 | `BranchesView.tsx` | 可用 | 本地分支列表、创建、切换、删除 |
| 远程 | `RemoteView.tsx` | 可用 | 远程列表、添加/删除远程、fetch/pull/push、保存项目级 token、Clone 远程仓库、远程配置聚合视图 |
| 安全 | `SafetyView.tsx` | 可用 | Git 用户名/邮箱/远程 URL、公共 token、SSH key 路径保存与掩码展示 |
| 自动 | `AutoView.tsx` | 占位 | 展示"自动备份"方向,尚未接定时提交、文件监听、规则配置 |

### 后端能力

`gt-git` 当前已经实现:

| 能力域 | 代码位置 | 已实现接口 |
| --- | --- | --- |
| 仓库生命周期 | `repo.rs` | `open`, `init`, `is_repo`, `workdir`, `overview` |
| 状态 | `status.rs` | 当前分支、工作区状态、文件状态结构 |
| diff | `diff.rs` | 工作区文件 diff、提交文件 diff |
| 提交 | `commit.rs` | `stage_all`, `stage_files`, `commit`, `commit_with_identity` |
| 分支 | `branch.rs` | 本地分支列表、创建、切换、删除 |
| 历史 | `log.rs` | 提交历史、分支历史、提交文件列表 |
| 远程 | `remote.rs` | 远程列表、添加、修改 URL、删除、fetch、push、fast-forward pull、`check_remote_access`、`clone_remote_repo_into_parent` |
| 认证 | `remote.rs` + Tauri command | Token、SSH key、SSH agent、None 的认证回调 |
| Pages 发布 | `pages.rs` | `prepare_pages_git`(分支准备)、`commit_pages_git`(提交+推送)、`verify_pages_push_access` |
| 错误 | `error.rs` | 统一错误类型 |

Tauri command 层在 `src-tauri/src/lib.rs` 暴露了这些能力,并在打开仓库、切换分支后调用 `store.sync_workspace(...)` 同步当前 workspace 状态。

### 已知缺口

- 还没有事件总线,Git 操作完成后前端仍主要靠手动 refresh 或视图初始化读取（commit 已发出 `git.commit.created` CloudEvent）。
- 还没有文件监听、定时自动提交、自动 commit message、危险操作前快照。
- `pull` 只支持 fast-forward,遇到分叉会报错并要求手动合并。
- 历史数据挖掘能力还未实现,例如文件最后修改时间 map、提交频率、文件提交次数。
- Git 二级视图已经注册表化,但还不是外部插件动态加载。

- 左侧列出所有命名空间、条目数量、public/private 可见性。
- 支持 Profile/环境 下拉切换。
- 右侧查看命名空间下所有 KV 条目。

### 已知缺口

- `StorePanel` 目前没有新增/编辑 KV 的 UI。
- Profile/环境前端已有创建/删除入口（通过环境别名命令）。
- `useStoreKey` 已存在,但还没有配合 Tauri event 做真正的响应式订阅。
- private 数据目前通过命名空间分层和掩码展示保护,尚未实现加密存储。

插件抽象当前是"静态注册表":

```text
PluginDescriptor
  id
  name
  description
  icon
  panel
  category
  pinned
```

已经能支撑内部模块扩展,但还不是 manifest、WASM 或外部包形式的插件系统。

## 建议下一步

1. 补齐数据中心前端写操作:新增/编辑 KV、同步状态展示面板。
2. 做 Git 自动模块 M1:文件监听、定时检测 dirty、生成固定格式 commit message、commit 后可选 push。
3. 做 store 事件推送:后端 `store_set` 后 emit,前端 `useStoreKey` 监听,减少手动 refresh。
4. 为复习模块补 Git 数据挖掘接口:按文件查询最后修改时间、提交次数、活跃度。
5. 把危险操作统一加确认与快照策略,例如删除分支、pull 前 dirty 检查、push 前敏感信息检查。
6. Flow 实际触发源接入:schedule 定时器、file_watch 文件监听、store_changed 配置变化。
