# GitTributary 插件规范

> 插件指由 manifest 发现并可运行时安装、更新、卸载的扩展。Git、Flow、数据、文件管理等
> `src/core/` 模块及其核心包不属于插件。

## 目录

```text
plugins/<plugin-name>/
├── manifest.json
├── frontend/
│   ├── package.json
│   ├── src/
│   └── dist/
└── backend/               # 可选 Rust 后端
    ├── Cargo.toml
    ├── src/
    └── target/release/
```

manifest 至少声明插件 ID、版本、页面入口、权限，以及可选的 `rust-cdylib` 后端。
插件应通过顶层 `icon` 声明市场与侧边栏共用的图标：

```json
{
  "icon": "lucide:paperclip"
}
```

当前内置图标注册表支持 `paperclip`、`send`、`bot`、`chart`、`code`、`database`、
`globe`、`search`、`settings`、`shield`、`terminal`、`wand`、`workflow`。视图未声明
`contributes.views[].icon` 时继承插件图标；视图级图标可用于覆盖默认值。未声明或宿主
暂不识别的图标回退为通用插件图标。

Rust backend 的每个公开方法必须在 `backend.methods` 中声明所需权限，宿主校验后才会
通过 `backend.invoke` 进入 sidecar。

## 开发

```bash
npm install
npm run tauri -- dev
```

Debug 命令会自动构建 sidecar 和全部插件。也可单独执行：

```bash
npm run plugin:build
```

新增插件只需放入 `plugins/`。构建脚本扫描 `plugins/*/manifest.json`；插件前端属于
`plugins/*/frontend` npm workspace，并提供 `build` 脚本。

## 构建与运行

```text
Debug   -> 项目 plugins/
Release -> 安装包 Resources/plugins/
安装后  -> ~/.git-tributary/plugins/<id>/<version>/
```

正式构建会收集 manifest、前端产物和当前平台动态库。卸载只删除用户安装目录，
不修改安装包内的插件资源，因此仍可再次安装。

## 能力边界

插件页面通过受控 Extension API 使用 Core 能力，目前开放：

- `repository:read`
- `files:read`
- `git:read`
- `git:write`
- `flow:read`
- `store:read` / `store:write`
- `shell:open`

插件页面不能直接调用任意 Tauri command。当前前端优先 React、后端优先 Rust；
不支持远程下载和第三方发布，原生 Rust ABI 也不视为稳定公开协议。

`plugins/site-publisher` 是插件示例：页面通过 Extension API 复用仓库、Store、
文件和系统打开能力。通用文件访问使用 `files.list`、`files.scan`、`files.search`、
`files.readText`；Markdown 候选筛选、站点渲染和 Pages 发布由插件 Rust backend 实现。
发布所需凭据由宿主在 `backend.invoke` 内部注入，不返回 iframe 页面。
