# 安全政策

Git Tributary 会处理本地 Git 仓库、remote 配置、发布目标仓库、数据中心同步配置和用户凭据。安全问题请谨慎报告，避免公开泄露 token、私钥、私有仓库地址或笔记内容。

## 支持版本

| 版本 | 安全支持状态 |
| --- | --- |
| 0.1.x | 支持 |
| 更早版本 | 不支持 |

## 报告安全问题

如果你发现安全问题，请优先使用 GitHub 的私密安全报告能力：

1. 打开仓库的 Security 页面。
2. 选择 Report a vulnerability 或创建私密安全公告。
3. 描述影响范围、复现步骤、受影响版本和建议修复方向。

如果仓库暂未开启私密安全报告，请创建一个最小化公开信息的 GitHub Issue，标题使用：

```text
[Security] Brief issue summary
```

请不要在公开 Issue 中粘贴：

- GitHub token、访问令牌或私钥。
- 私有仓库地址。
- 个人笔记内容。
- 可直接利用的完整攻击细节。

## 报告内容建议

请尽量提供：

- Git Tributary 版本号。
- 操作系统和芯片架构。
- 受影响模块，例如 Git、发布、Flow、数据中心或凭据。
- 最小复现步骤。
- 你认为可能造成的影响。
- 已经采取的缓解措施。

## 安全范围

重点关注：

- token、SSH key、凭据引用或私有配置泄露。
- 私有笔记被错误发布到公开 Pages 仓库。
- 数据中心同步错误地包含 private 命名空间。
- Git remote 操作绕过用户确认或写入错误仓库。
- Flow 执行过程中出现越权文件读写或非预期命令执行。
- DMG、更新包、校验文件或发布资产被篡改。

## 用户侧安全建议

- 从官方 GitHub Release 下载安装包。
- 校验 Release 中提供的 SHA256 摘要。
- 优先安装已签名和已公证的 macOS 发布包。
- 为 GitHub token 设置最小权限并定期轮换。
- 发布前检查 Markdown 范围和目标 Pages 仓库是否正确。
- 不要把 `~/.git-tributary/` 或含有 token 的配置目录提交到公开仓库。
